# FishNote — 구현 아키텍처

> 대상: 전체 구현 청사진. Codex 작업의 기준 문서.

---

## 1. 시스템 구성

```
[React (Vite/TS)]  ──REST(JSON)──>  [Spring Boot]  ──JPA──>  [PostgreSQL]
   Vercel 배포                         Railway 배포            Railway DB
        │                                   │
   axios + TanStack Query            /api/v1/** 엔드포인트
```

- FE와 BE는 **완전 분리**. FE는 BE의 REST API만 호출.
- API 기본 경로: `/api/v1`
- 응답 포맷: JSON. 엔티티 직접 반환 금지 → **DTO 변환**.

## 2. 스택 / 버전

## 4. 백엔드 계층 구조 (도메인별 패키지)

```
Controller  →  Service  →  Repository  →  Entity
   (REST)      (로직)       (JPA)         (DB매핑)
                 │
                 └→ DTO 변환 후 응답
```

- **Controller**: 요청/응답만. 검증(@Valid), DTO 반환.
- **Service**: 비즈니스 로직, 트랜잭션(@Transactional), 엔티티↔DTO 변환.
- **Repository**: Spring Data JPA 인터페이스. 필터 쿼리.
- **DTO**: 요청(`*Request`) / 응답(`*Response`) 분리. 엔티티 노출 금지.
- **common**: `@RestControllerAdvice` 전역 예외처리 + 표준 에러 응답.

도메인은 `fish`, `review` 두 개로 시작. 패키지를 도메인 기준으로 나눠 응집도를 높임.

## 5. 프론트엔드 구조

- **pages**: 라우트 단위 화면. `/`(목록) · `/fish/:id`(상세) · `/search`(검색결과)
- **components**: 재사용 UI. 디자인 시안의 카드/칩/폼을 컴포넌트로.
- **api**: axios 인스턴스(baseURL = 환경변수) + 도메인별 호출 함수.
- **hooks**: TanStack Query로 감싼 데이터 훅(`useFishList` 등) → 로딩/에러/캐싱 자동.
- **types**: 백엔드 DTO와 1:1 대응하는 TS 타입.

데이터 흐름: `page` → `useXxx 훅(react-query)` → `api 함수(axios)` → BE.

## 6. 로컬 개발

```bash
# DB (로컬 PostgreSQL 실행 후 fishnote DB 생성)
createdb fishnote

# 백엔드 (BE/)
./gradlew bootRun        # http://localhost:8080

# 프론트 (FE/)
npm install
npm run dev              # http://localhost:5173
```

- FE `.env`: `VITE_API_BASE_URL=http://localhost:8080/api/v1`
- BE `application.yml`: DB 접속정보 + `ddl-auto: update`(개발) / 시드는 `data.sql`

---

## 7. CORS / 환경변수 (중요)

- BE `CorsConfig`에서 FE 오리진 허용:
  - 로컬: `http://localhost:5173`
  - 운영: `https://<도메인>` (Vercel 주소 + 산 도메인)
- 민감정보(DB 비번 등)는 **환경변수**로. 코드/깃에 커밋 금지.
  - BE: `SPRING_DATASOURCE_URL/USERNAME/PASSWORD`
  - FE: `VITE_API_BASE_URL`

## 8. 배포 타깃

| 영역 | 플랫폼 | 메모 |
|---|---|---|
| DB | Railway PostgreSQL | 플러그인으로 생성, 접속정보 환경변수 주입 |
| BE | Railway | Gradle 빌드 → 컨테이너 실행, 8080 노출 |
| FE | Vercel | Vite 빌드, `VITE_API_BASE_URL`에 운영 BE 주소 |
| 도메인 | (구매 완료) | Vercel에 연결, BE는 서브도메인(api.도메인) 권장 |

## 9. 이미지 처리 (v1 단순화)
