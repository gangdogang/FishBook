# FishBook — 회 도감 (포트폴리오 · 풀스택 학습 프로젝트)

> 한 줄 정의: **회에 관심 있는 사람이 "내가 먹는/먹을 회가 어떤 생선인지, 언제가 제철인지, 맛은 어떤지"를 둘러보고 검색하는 회 도감.**
>
> 목표: **풀스택 개발자 포트폴리오 + 학습.** 빠른 배포보다 "제대로 만들고 이해하는 것" 우선. (배포는 학습 마일스톤의 마지막 단계)

---

## 1. 방향 · 타겟

- **주 사용자**: 회 입문자 · 마니아 · 횟집 손님 · 수산시장 구경객
- **핵심 가치**: 모르는 생선을 쉽게 찾아보고, 제철·맛·가격대를 한눈에
- **성공 기준**: ① 방문자 1,000명 ② 광고 수익화 ③ **포트폴리오로서 "FE/BE/DB/배포 전부 내가 했다"를 보여주기**
- **핵심 사용자 행동**: 둘러보기 + 검색하기

---

## 2. MVP 기능 범위

### 꼭 넣을 것 (v1)
1. **생선 둘러보기** — 카드 목록 (사진 + 이름 + 제철 뱃지)
2. **생선 상세 페이지** — 사진, 제철, 맛, 가격대, 비슷한 생선
3. **검색 / 필터** — 이름 검색 + 제철(계절) 필터 + 맛 태그 필터
4. **횟감 리뷰(후기)** — 생선별 후기 작성/조회

### 뺄 것 (나중에)
- 지도 / 가성비 횟집 찾기 (v2 핵심 후보)
- 정식 로그인은 v1 후반(인증 학습 단계)에 추가 → 초반엔 익명 닉네임으로 시작
- 즐겨찾기, 알림

---

## 3. 아키텍처 (풀스택 · 분리형)

```
[React (FE)] ──REST API──> [Spring Boot (BE)] ──JPA──> [PostgreSQL]
   Vercel 배포               Railway/Render 배포          관리형 DB
```

| 영역 | 선택 | 학습 포인트 |
|---|---|---|
| 프론트엔드 | **React + Vite + TypeScript** | 컴포넌트 설계, 상태관리, API 연동, 반응형 |
| 스타일 | **Tailwind CSS** | 빠른 미니멀 UI |
| 백엔드 | **Spring Boot (Java)** | REST API 설계, 계층 구조(Controller-Service-Repository) |
| DB 접근 | **Spring Data JPA** | 엔티티 매핑, 연관관계, 쿼리 |
| DB | **PostgreSQL** | 스키마 설계, 관계형 모델링 |
| 인증 | **Spring Security + JWT** (후반) | 인증/인가 개념 |
| 배포 | FE: Vercel / BE+DB: Railway 또는 Render | CORS, 환경변수, CI/CD |

> 폴더 구조: `FishBook/FE` = React, `FishBook/BE` = Spring Boot. git은 루트에서 모노레포로 관리.

## 4. 데이터 스키마

### `fish` (생선)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | Long (PK) | |
| name | String | 이름 (예: 광어) |
| name_en | String | 영문/학명 (선택) |
| image_url | String | 대표 사진 |
| season_months | int[] / 별도 테이블 | 제철 월 (예: 11,12,1,2) |
| taste_tags | String[] / 별도 테이블 | 맛 태그 (담백, 기름진, 쫄깃 등) |
| taste_desc | String | 맛 설명 |
| price_level | int | 가격대 (1~3) |
| description | String | 한 줄 소개 |
| created_at | timestamp | |

### `similar_fish` (비슷한 생선 — 다대다)
| fish_id | similar_id | (fish 끼리 연결) |

### `review` (후기)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | Long (PK) | |
| fish_id | Long (FK) | → fish |
| nickname | String | 작성자 |
| rating | int | 별점 1~5 (선택) |
| content | String | 후기 |
| image_url | String | 사진 (선택) |
| created_at | timestamp | |

> JPA 학습 포인트: fish ↔ review = **1:N**, fish ↔ similar_fish = **N:M(self)**. 연관관계 매핑 연습에 딱 좋음.

## 5. 학습 로드맵 (단계별, 데드라인 없음)

### Phase 0 — 준비 & 개념 잡기
- Java/Spring 기본, REST API가 뭔지, 클라이언트-서버 흐름 이해
- 개발 환경: JDK, IntelliJ(BE), VS Code(FE), PostgreSQL 로컬 설치

### Phase 1 — 백엔드 먼저 ⭐ (가장 많이 배움)
- Spring Boot 프로젝트 생성 (Spring Web, JPA, PostgreSQL 의존성)
- DB 스키마 → JPA 엔티티 작성 (fish, review)
- `GET /fish`, `GET /fish/{id}` REST API 구현
- Postman으로 API 테스트, 생선 5종 더미 데이터 입력

### Phase 2 — 프론트엔드 화면
- React + Vite 셋업, Tailwind 적용
- 둘러보기 목록 + 상세 페이지 UI
- 백엔드 API 연동 (fetch/axios), CORS 처리 경험

### Phase 3 — 검색 · 필터 · 리뷰
- 이름 검색 + 제철/맛 필터 API + 화면
- `POST /fish/{id}/reviews` 리뷰 작성 API + 폼
- 리뷰 목록 조회

### Phase 4 — 인증 (선택 심화)
- Spring Security + JWT로 로그인/회원가입
- 인증/인가 개념 학습

### Phase 5 — 배포
- BE + PostgreSQL → Railway/Render 배포
- FE → Vercel 배포, 산 도메인 연결
- 환경변수/CORS/HTTPS 마무리

### Phase 6 — 콘텐츠 & 다듬기
- 생선 10~20종 채우기 (사진 저작권 주의: 공개 자료 출처 확인 + 직접 촬영)
- 모바일 반응형, 디자인 다듬기, 출시
