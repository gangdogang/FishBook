FISHNOTE · 회 도감 · 디자인 제안 v1

모르는 생선 앞에서 3초 — 누구나 쓰는 회 도감

기획.md와 API 명세(v1 확장)를 근거로, 기존 FE 화면은 보지 않고 처음부터 다시 그린 제안입니다. 타겟은 회 입문자·횟집 손님·수산시장 구경객 — 전문 지식 없이도 "이 생선 뭐지? 지금 제철인가?"에 바로 답을 주는 것이 이 디자인의 유일한 목표입니다.

① 디자인 원칙

세 가지만 지킵니다

기능이 이미 잘 정리되어 있으므로(도감·검색·캘린더·후기), 디자인은 더하지 않고 정리하는 역할만 합니다.

"지금 제철"이 첫 번째 정보

색으로 강조하는 건 딱 하나 — 지금 제철 뱃지와 주요 버튼뿐. 나머지 정보(맛·가격·별점)는 회색 계열로 조용히 받쳐줍니다. 강조가 하나면 눈이 갈 곳이 명확해집니다.

전문용어 제로

월 배열 [11,12,1,2]는 "제철 11–2월"로, 가격 레벨 1~3은 "₩ 부담 없는 / ₩₩ 보통 / ₩₩₩ 특별한 날"로. 데이터를 사람의 말로 번역하는 것이 이 앱 디자인의 절반입니다.

한 화면, 한 가지 일

홈은 찾기, 상세는 알기, 캘린더는 고르기. 화면마다 목적을 하나로 좁히고 그 외 요소는 뺍니다. 히어로·배너·팝업 같은 장식성 요소는 넣지 않습니다.

② 파운데이션

바닷빛 하나, 나머지는 안개색

포인트 컬러는 "물빛" 청록 하나입니다. 새벽 수산시장의 물빛에서 가져왔고, 신선함·신뢰를 동시에 줍니다. 별점의 호박색은 별점 전용 의미 색이라 포인트로 치지 않습니다.

기존 구조에 그대로 얹기

API·라우트·컴포넌트 구조는 손대지 않습니다. 아래는 기존 FE 파일과 이 디자인의 대응표입니다.

기존 파일이 디자인에서
AppLayout.tsx헤더 3메뉴(도감·제철 캘린더·저장한 도감). 홈에서는 헤더 검색 숨기고 히어로 검색만
FishCard.tsx사진 4:3 고정 크롭 · "지금 제철" 뱃지는 seasonMonths.includes(현재 월) 클라 계산 · 하트 토글
SeasonBar.tsx12칸 바 + "지금" 마커 (상세 전용, 카드에서는 "제철 6–8월" 텍스트 뱃지)
MonthSidebar.tsx가로 12칸 월 셀렉터로 교체 제안 — 모바일에서 가로 스크롤 칩
FilterChips.tsx계절·맛·가격대 3개 드롭다운 칩 + 선택 시 물빛 채움/✕ 해제 + 초기화
SearchBar.tsx홈 히어로용(대) / 서브페이지 헤더용(소) 두 사이즈
ReviewForm.tsx4개 필드 상시 노출, 비밀번호 헬퍼 문구, 사진 URL 입력은 v1에서 제외 제안
ReviewList.tsx요약(평균+분포) 좌측 고정 + 최신순/도움순 칩 + 도움돼요 "눌렀어요" 상태

  • API 추가 요청 없음. 홈 = ?month=N + ?featured=true + 기본 목록, 캘린더 = ?month=N, 후기 = sort=latest|helpful — 전부 명세 v1 확장 범위 안입니다.
  • 다크 모드는 v1 범위 밖. 사진 중심 서비스라 라이트 단일 테마로 시작하고, 반응 좋으면 v2에서 검토.
  • 적용 순서 제안: ① 토큰·타이포 → ② FishCard·뱃지·칩 → ③ 홈 → ④ 상세 → ⑤ 캘린더·저장 → ⑥ 상태·카피.

⑥ 진행 전 확인하고 싶은 것

세 가지만 답해주세요

  1. 서비스 이름 — 폴더는 FishBook, 문서·코드는 FishNote입니다. 로고 워드마크를 확정해야 해서, 어느 쪽으로 갈까요?
  2. 생선 사진 수급 계획 — 직접 촬영인지, 공개 자료인지에 따라 사진 톤 보정 가이드(4:3 크롭 + 밝기 통일)가 달라집니다. 사진이 없는 생선은 위 목업처럼 일러스트 플레이스홀더로 대체하는 방식을 제안합니다.
  3. 사진 URL 후기 첨부 — API에는 있지만 입문자에게 URL 입력은 장벽이라 v1 화면에서 뺐습니다. 유지할까요, 뺄까요? (v2 파일 업로드 때 다시 넣는 것을 추천)
