# FishBook — 디자인 이식 티켓 (HTML 시안 → React)

> 전제: 클로드로 만든 디자인 HTML을 `docs/design-reference.html`(또는 화면별 파일)로 저장해둔다.
> 핵심 원칙: **겉모습(마크업·Tailwind 클래스·색·간격·폰트)은 시안대로 이식, 데이터·API 연동·라우팅·상태관리는 기존 코드 그대로 유지.** 기능 로직은 건드리지 않는다.
> 진행 순서: D0(토큰) → D1(공통/카드) → D2(목록) → D3(상세) → D4(검색) → D5(후기) → D6(검수).

---

## D0 — 디자인 토큰 동기화
**프롬프트**:
> `docs/design-reference.html`에서 사용한 색·폰트·라운드·그림자 값을 추출해서 `FE/tailwind.config.js`의 theme(colors, fontFamily, borderRadius 등) 토큰과 일치시켜줘. 시안과 코드의 Teal 등 색상값이 다르면 시안 기준으로 맞춰줘. 전역 폰트(Pretendard 등)가 시안에 있으면 `index.css`에 적용. 기존 클래스 사용처가 깨지지 않게 토큰명은 유지.

**DoD**: tailwind 토큰이 시안 색/폰트와 일치, 빌드 통과.

---

## D1 — 공통 레이아웃 + FishCard
**프롬프트**:
> `docs/design-reference.html`의 헤더(로고/네비), 전체 레이아웃 컨테이너, 그리고 생선 카드 디자인을 기준으로 `FE/src/components/FishCard.tsx`와 공통 레이아웃(헤더/푸터)을 시안과 똑같이 스타일링해줘. **props·데이터 구조·onClick 라우팅은 기존 그대로 두고**, className/마크업만 시안에 맞춰. 카드에 들어가는 항목(사진·이름·제철뱃지·소개·맛태그·가격대·별점)은 기존 데이터 필드를 그대로 사용.

**DoD**: 카드·헤더가 시안과 동일한 모양, 데이터는 실제 API 값 유지.

---

## D2 — 둘러보기(목록) HomePage
**프롬프트**:
> `docs/design-reference.html`의 목록 화면을 기준으로 `FE/src/pages/HomePage.tsx`, `SearchBar.tsx`, `FilterChips.tsx`를 시안대로 스타일링해줘. 검색·필터 칩의 활성/비활성 상태 스타일을 시안대로. **useFishList 쿼리 연동과 필터 상태 로직은 기존 유지**, 마크업·클래스만 교체.

**DoD**: 목록 화면이 시안과 일치, 필터·검색·카드 클릭 정상 동작.

---

## D3 — 상세 FishDetailPage
**프롬프트**:
> `docs/design-reference.html`의 상세 화면을 기준으로 `FishDetailPage.tsx`, `SeasonBar.tsx`를 시안대로 스타일링해줘. 대표사진·이름(국/영)·제철뱃지·가격대·별점·맛 설명/태그·제철 시각화·비슷한 생선 카드·후기 섹션 레이아웃을 시안에 맞춰. **useFishDetail/useReviews 연동은 그대로 유지.**

**DoD**: 상세 화면이 시안과 일치, 비슷한 생선/후기 정상 표시.

---

## D4 — 검색 결과 SearchPage
**프롬프트**:
> `docs/design-reference.html`의 검색/필터 결과 화면을 기준으로 `SearchPage.tsx`를 시안대로 스타일링해줘. 필터 영역·결과 개수·정렬·결과 그리드·빈 상태(empty state) 디자인 반영. **URL 쿼리 동기화 로직은 기존 유지.**

**DoD**: 검색 화면·빈 상태가 시안과 일치, 필터 동작 유지.

---

## D5 — 후기 ReviewList / ReviewForm
**프롬프트**:
> `docs/design-reference.html`의 후기 목록·작성 폼(모달) 디자인을 기준으로 `ReviewList.tsx`, `ReviewForm.tsx`를 시안대로 스타일링해줘. 별점 UI, 입력 필드, 버튼, 에러 메시지 스타일 반영. **작성/삭제 mutation·검증 로직은 그대로 유지.**

**DoD**: 후기 UI가 시안과 일치, 작성/삭제 정상 동작.

---

## D6 — 시각 검수
- `npm run dev`(5173)로 모든 화면을 시안과 나란히 비교.
- 어긋난 부분(간격·색·정렬·반응형)만 모아서 Codex에 추가 요청.
- 모바일 폭(예: 375px)에서도 깨지지 않는지 확인.

## 작업 팁
- **화면 하나씩** 이식 → 매번 브라우저로 확인 → 다음 화면. 한 번에 다 하면 디버깅이 어려움.
- 시안과 실제 데이터(글자 길이·이미지 비율)가 달라 깨지는 부분은 시안이 아니라 실데이터 기준으로 조정.
- 기능이 깨졌다 싶으면 "스타일만 바꾸고 로직은 유지"를 다시 강조해서 재요청.
