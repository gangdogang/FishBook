# FishNote — 리디자인 구현 설계 (Codex용)

> 시안: `docs/design-reference-v2.html` — **`.screen` 스코프 안의 `--p-*` 토큰과 마크업이 제품 기준**. 문서 래퍼(제안서 껍데기) 스타일은 무시할 것.
> 원칙: **기능 로직(API 연동·라우팅·상태관리)은 그대로, 스타일·마크업·카피만 교체.** 예외는 신규 기능 2건(U1 사진 업로드, R5 캘린더 월 셀렉터 교체)뿐.
> 서비스명은 **FishNote로 확정** — 화면에 보이는 모든 브랜드 표기를 통일한다.
> 순서: R0(토큰) → R1(공통 컴포넌트) → R2(레이아웃) → R3(홈) → R4(상세) → R5(캘린더·저장) → R6(후기) → U1·U2(사진 업로드) → R7(상태·카피) → R8(검수).

## 0. 디자인 3원칙 (모든 티켓의 판단 기준)

1. **"지금 제철"만 색으로 강조** — 포인트 컬러(물빛)는 지금 제철 뱃지·주요 버튼·링크에만. 나머지는 중성색.
2. **전문용어 제로** — 월 배열은 "제철 6–8월", priceLevel은 "₩₩ 보통"처럼 항상 사람의 말로 번역해서 표시.
3. **한 화면, 한 가지 일** — 홈=찾기, 상세=알기, 캘린더=고르기. 장식성 요소(배너·팝업) 추가 금지.

## R0 — 디자인 토큰 + 폰트

**프롬프트**:
> `FE/tailwind.config.js`의 theme.extend를 아래 토큰으로 교체하고, `npm i pretendard` 후 `main.tsx`에서 `import 'pretendard/dist/web/variable/pretendardvariable-dynamic-subset.css'`를 추가해줘. `index.css`의 body 기본값: `bg-mist text-ink`, font-size 15px, line-height 1.7. 기존 클래스 사용처가 깨지지 않게 빌드 확인.

```js
// tailwind.config.js — theme.extend
colors: {
  sea:  { DEFAULT: '#0F6E84', soft: '#E3F0F3' }, // 물빛(유일한 포인트)
  ink:  { DEFAULT: '#1A2B33', mute: '#5F7078' }, // 텍스트 / 보조 텍스트
  mist: '#F6F8F9',                               // 페이지 배경
  line: '#E3EAED',                               // 경계선
  chipbg: '#EEF3F5',                             // 칩 배경
  star: '#E0A030',                               // 별점 전용(포인트 아님)
},
borderRadius: { card: '14px' },
fontFamily: { sans: ['Pretendard Variable','Pretendard','Apple SD Gothic Neo','Noto Sans KR','sans-serif'] },
```

**타이포 스케일**: 디스플레이 28/800 · 섹션 타이틀 19/800 · 본문 15/400 · 캡션 13 · 라벨(뱃지·칩) 12/700. 숫자가 나오는 곳은 `tabular-nums`.

**DoD**: 토큰 적용·Pretendard 렌더·빌드 통과.

## R1 — 공통 컴포넌트 스펙

**프롬프트**:
> 시안 `.screen` 스코프 기준으로 아래 스펙대로 공통 컴포넌트를 스타일링/작성해줘. props와 데이터 필드는 기존 그대로 사용.

### 제철 뱃지 (상태 2개뿐)
- **지금 제철**: `seasonMonths.includes(현재 월)`이면 표시. bg `sea` / 흰 글자 12/700 / rounded-full / px 10 py 3 / 앞에 5px 흰 점. **물빛을 채워 쓰는 유일한 뱃지.**
- **제철 기간**: 흰 배경 + `line` 테두리 + `ink-mute` 12/600. 라벨 규칙(`lib/format.ts`에 유틸):
  - 연속 구간 → `제철 6–8월`, 12개월 전부 → `연중`, 겨울처럼 해를 넘는 경우 12→1 연결 판정 → `제철 11–2월`.

### 가격대
- `'₩'.repeat(priceLevel)` + 라벨 `{1: '부담 없는', 2: '보통', 3: '특별한 날'}`.
- 카드에서는 기호만(13/700), 상세·필터에서는 `₩₩ 보통`처럼 기호+라벨(라벨은 `ink-mute`).

### 별점
- 표기: `★ 4.3 (12)` — star 색 ★ 1개 + 평균 소수 1자리 + 회색 괄호 후기 수. 아이콘 단독 사용 금지.
- 후기 0개: 카드에서는 별점 영역 생략, 상세에서는 "아직 후기가 없어요".

### 맛 태그 / 필터 칩
- 기본: bg `chipbg` / `ink` 13/600 / rounded-full / px 13 py 5.
- 선택(필터): bg `sea` / 흰 글자 + `✕`로 해제. 드롭다운 트리거 칩에는 `▾`.

### SeasonBar (상세 전용)
- 12칸 그리드, 칸 높이 8px rounded gap 3px. 제철 월 = `sea` 채움, 나머지 `chipbg`.
- 현재 월 칸 위에 "지금" 10px/700 `sea` 마커, 칸 아래 월 숫자 10px `ink-mute`.

### FishCard
- 흰 배경 / `line` 1px 테두리 / rounded-card / 그림자 없음(호버 시만 살짝).
- 이미지 4:3 고정(`aspect-ratio` + `object-cover`), 좌상단(10px) 지금 제철 뱃지, 우상단(8px) 저장 하트 — 32px 원형, bg `white/90`, 저장 시 `sea` 채운 하트.
- 본문(p-3.5): 이름 16/700 + 별점 우측 정렬 → 한 줄 소개 13 `ink-mute`(1줄 ellipsis) → 메타 행(제철 기간 뱃지 좌 / 가격 우).

**DoD**: 스토리 화면 또는 홈에서 각 상태(지금 제철·저장됨·선택 칩) 확인, 실데이터 렌더 정상.

## R3 — 홈 (HomePage)

구성 순서(위→아래), 데이터는 전부 기존 API:

| 섹션 | 내용 | 데이터 |
|---|---|---|
| ① 검색 히어로 | 중앙 정렬. "오늘 이 회, 무슨 생선일까요?" 28/800 + 서브 "이름·제철·맛·가격대까지, 3초면 확인해요" + 큰 검색창(max-w 520, 우측 `sea` 검색 버튼) + "많이 찾는 생선" 정적 태그 4개(광어·방어·연어·참돔, 클릭 시 검색) | 정적 |
| ② 이달의 제철 | 제목 "N월, 지금이 제철이에요"(N=현재 월 자동) + 우측 "제철 캘린더 →" 링크. 카드 4개 그리드(모바일 가로 스와이프) | `GET /fish?month=N` |
| ③ 에디터 추천 | 낮은 가로형 카드 2장(이미지 aspect 5:2), 소개 문구를 추천 이유처럼 노출 | `GET /fish?featured=true` 앞 2개 |
| ④ 전체 도감 | 제목 + 종 수. 필터 행: 칩 3개(계절·맛·가격대 드롭다운) + 우측 정렬 세그먼트(인기순/이름순). 카드 4열 그리드(태블릿 2열, 모바일 1열) | `GET /fish` + 기존 필터 상태 |

**프롬프트**:
> 위 표와 시안 기준으로 `HomePage.tsx`, `SearchBar.tsx`, `FilterChips.tsx`를 스타일링해줘. useFishList 훅·필터 상태 로직은 그대로 유지하고 마크업/클래스만 교체. 히어로 검색은 제출 시 기존 `/search` 흐름 그대로. 선택된 필터 칩은 `sea` 채움 + ✕ 해제, "필터 초기화" 텍스트 버튼 포함.

**DoD**: 4개 섹션 렌더, 이달의 제철이 현재 월 자동 반영, 필터·검색·카드 클릭 정상.

## R6 — 후기 UI (ReviewList / ReviewForm)

레이아웃: 좌 250px 요약 패널 + 우 목록(모바일 1열, 요약이 위).

- **요약 패널**: 평균 `4.8 / 5` 34/800 → 별 행 → "후기 6개" → 별점 분포 5→1 (라벨 `5점` / 막대 track `chipbg`·fill `sea` h-6px / 개수). 데이터 `ratingDistribution`.
- **정렬**: 칩 2개 `최신순`(기본) / `도움순` → `sort=latest|helpful`.
- **후기 카드**: 아바타(32px 원형 `sea-soft` 배경 + 닉네임 첫 글자 `sea`) + 닉네임 14/700 + 별점 + 날짜 12 `ink-mute`, 우측 끝 "삭제" 12 `ink-mute`(클릭 → 비밀번호 입력 모달 → DELETE). 본문 14. 사진 있으면 썸네일(U2). 하단 **도움돼요 필**: 기본 `👍 도움돼요 4` / 누른 후 `👍 도움돼요 5 · 눌렀어요`(`sea` 테두리+`sea-soft` 배경, 비활성). 중복 방지 localStorage `helpful:<reviewId>`.
- **작성 폼**(목록 하단, 모달 아님 — 항상 펼쳐진 카드): 닉네임 / **별점 (선택)** 별 5개 탭 / 후기 textarea(placeholder "맛·식감·먹은 곳 분위기, 자유롭게 적어주세요") / 비밀번호 + 헬퍼 "후기를 지울 때만 써요 (4자 이상)" / `등록하기` primary. **이미지 URL 텍스트 입력은 제거**(U2의 파일 첨부로 대체). 검증 에러는 해당 필드 아래 13px로.

**프롬프트**:
> 위 스펙과 시안대로 `ReviewList.tsx`, `ReviewForm.tsx`를 구현/스타일링해줘. 작성·삭제 mutation과 invalidate 로직은 기존 유지. 도움돼요는 `POST /reviews/{id}/helpful` 연동 + localStorage 1회 제한.

**DoD**: 분포·정렬·도움돼요(1회 제한) 동작, 폼 검증 메시지 표시, URL 입력란 제거됨.

## U1 — BE 이미지 업로드 API (신규)

> 배경: 후기 사진은 사용자가 폰으로 찍어 올린다. **사용자는 URL을 보지도 입력하지도 않는다** — 파일을 고르면 서버가 이미지 호스팅(Cloudinary)에 올리고 URL을 만들어 기존 `imageUrl` 필드에 저장하는 구조.
> Railway 컨테이너는 재배포 시 디스크가 초기화되므로 서버 디스크 저장은 금지. 외부 스토리지 필수.
> 사전 준비(사람이 할 일): Cloudinary 무료 계정 생성 → `CLOUDINARY_URL` 환경변수 확보(형식 `cloudinary://<api_key>:<api_secret>@<cloud_name>`).

**프롬프트**:
> Spring Boot에 이미지 업로드 API를 추가해줘. `POST /api/v1/images` (multipart/form-data, 필드명 `file`). 의존성 `com.cloudinary:cloudinary-http5` 추가, 환경변수 `CLOUDINARY_URL`로 초기화하는 `config/CloudinaryConfig` 작성. 검증: 파일 필수, 최대 5MB, content-type `image/*`만 허용(위반 시 표준 400 에러). 업로드는 Cloudinary 폴더 `fishnote/reviews`, 응답은 `{ "url": "https://res.cloudinary.com/..." }` 201. `application.yml`에 `spring.servlet.multipart.max-file-size: 5MB` 설정. 전역 예외처리(§04명세 §7 포맷) 적용.

**DoD**: curl 파일 업로드 → Cloudinary URL 반환, 5MB 초과·비이미지 파일 400, 키는 환경변수로만 주입.

## U2 — FE 후기 사진 첨부 (신규)

**프롬프트**:
