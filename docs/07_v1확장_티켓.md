# 백엔드 확장

## B1 — featured + month 필터
**참고**: `04_API명세.md §9-1`
**프롬프트**:
> `fish`에 `featured`(boolean, 기본 false) 컬럼을 추가하고, `GET /fish`에 `month`(1~12, 해당 월 제철 생선)과 `featured`(true면 EDITOR'S PICK만) 파라미터를 추가해줘. `FishSummaryResponse`에 `featured` 필드 포함. 기존 필터(search/season/taste/priceLevel/sort)와 조합 가능해야 함.

**DoD**: `?featured=true`·`?month=12` 동작, summary에 featured 노출.

---

## B2 — 다중 이미지
**참고**: `01_DB설계.md §7`, `04_API명세.md §9-2`
**프롬프트**:
> `fish_image` 컬렉션(@ElementCollection + @OrderColumn)을 추가해 생선 갤러리 이미지를 순서대로 저장해줘. `FishDetailResponse`에 `images`(대표 포함 순서 배열) 추가. 이미지가 없으면 `[imageUrl]` 반환.

**DoD**: 상세 응답에 images 배열 정상 반환.

---

## B3 — tips
**참고**: `01_DB설계.md §7`, `04_API명세.md §9-2`
**프롬프트**:
> `fish_tip` 컬렉션(@ElementCollection + @OrderColumn, content)을 추가해 "이렇게 즐겨요" 팁을 순서대로 저장해줘. `FishDetailResponse`에 `tips`(문자열 배열) 추가.

**DoD**: 상세 응답에 tips 배열 정상 반환.

---

## B4 — 도움돼요 + 별점 분포
**참고**: `04_API명세.md §9-3, §9-4`
**프롬프트**:
> `review`에 `helpful_count`(int, 기본 0)를 추가하고 `POST /reviews/{id}/helpful`(카운트 +1, 새 값 반환)을 구현해줘. `GET /fish/{id}/reviews`에 `sort=latest|helpful` 정렬과 응답 `ratingDistribution`(별점 1~5 개수), 각 리뷰의 `helpfulCount`를 추가. `FishDetailResponse`에도 `ratingDistribution` 추가.

**DoD**: 도움돼요 증가 동작, 분포·정렬·helpfulCount 정상.

---

## B5 — 시드 데이터 보강
**프롬프트**:
> `data.sql`을 보강해줘. 기존 6종에 featured 플래그(2~3종 true), 생선별 tips 2개, 갤러리 이미지 2~3장(임시 URL), 그리고 일부 생선에 더미 후기 2~3개(별점 다양, helpful_count 포함)를 추가해 화면이 비어 보이지 않게.

**DoD**: 기동 후 에디터's pick·tips·갤러리·후기·별점분포가 채워진 상태.

# 프론트 (이식 중 함께)

## D0~D6 — 디자인 이식
`06_디자인_이식.md`의 D0~D6 그대로 진행하되, 새 기능 반영:
- D2(목록): 히어로(인기 검색 태그=정적), EDITOR'S PICK 섹션(`?featured=true`), 전체 도감
- D3(상세): 갤러리 썸네일(images), spec grid, season bar, tips, 별점 분포
- D5(후기): helpfulCount("도움돼요" 버튼 → `POST /reviews/{id}/helpful`), 별점 분포, latest/helpful 정렬

## B6 — 저장 북마크 (프론트 localStorage)
**참고**: `04_API명세.md §9-5`
**프롬프트**:
> 생선 저장(북마크)을 localStorage로 구현해줘. 카드/상세의 저장 버튼 토글, 저장 상태 아이콘 반영. 헤더의 "저장한 도감"에서 저장된 fish id 목록을 `GET /fish` 결과에서 필터링해 보여줘. 로그인 없이 동작.

**DoD**: 저장/해제 토글, 새로고침 후 유지, 저장한 도감 목록 표시.

## B7 — 제철 캘린더 화면
**참고**: `04_API명세.md §9-1`(month)
**프롬프트**:
> 시안의 "제철 캘린더" 화면을 `/calendar` 라우트로 구현해줘. 월(1~12) 선택 버튼, 선택 월의 제철 생선을 `GET /fish?month=N`으로 조회해 카드 그리드로 표시. 각 월 버튼에 해당 월 제철 생선 수 표시(선택 월 데이터로 갱신). 헤더 네비에 "제철 캘린더" 추가.

**DoD**: 월 선택 시 해당 제철 생선 표시, 네비 이동 동작.
